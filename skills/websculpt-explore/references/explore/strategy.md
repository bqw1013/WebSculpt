# WebSculpt Explore 策略

> 本文档面向 AI Agent，说明面对信息获取任务时如何选择工具、复用已有 WebSculpt 命令、推进探索，并在结束时判断是否存在 capture 候选。

## 1. 总原则

探索的目标是先可靠完成用户当前请求，同时为后续 capture 留下足够证据。

- **先复用**：调用外部信息获取工具前，先检查本地 WebSculpt 命令库。
- **轻重分层**：公开静态信息优先使用轻工具；登录态、交互、JS 渲染或强反爬场景直接使用浏览器自动化。
- **一手信息优先**：搜索引擎用于定位来源，找到来源后尽量读取原始页面或 API。
- **及时止损**：同一工具多次调整仍无实质改善时，切换策略。
- **记录路径**：保留实际跑通的 URL、API、参数、选择器、输出结构和失败信号。

## 2. 第一步：检查命令库

WebSculpt 命令库保存已经沉淀的信息获取路径。每条命令通常覆盖一个小而稳定的子任务。

在调用 WebSearch、WebFetch、curl 或浏览器自动化前，先检查：

```bash
websculpt command list
```

若需要结构化输出：

```bash
websculpt --format json command list
```

发现候选命令后，按需确认适用性：

```bash
websculpt <domain> <action> --help
websculpt command show <domain> <action>
websculpt command show <domain> <action> --include-readme
```

确认命令覆盖当前子任务后，直接调用命令。若命令不覆盖、参数不适合或返回结果不足，再进入外部工具探索。

## 3. 第二步：选择外部工具

根据任务特征选择起点：

| 场景 | 优先工具 |
|------|---------|
| 需要发现信息来源或搜索摘要 | WebSearch |
| URL 已知，需要读取正文 | WebFetch / curl |
| URL 已知，需要原始 HTML、meta、JSON-LD | curl |
| 内容依赖登录态、JS 渲染或多步骤交互 | 浏览器自动化 |
| 目标站点已知静态抓取困难或有强反爬 | 浏览器自动化 |

若任务特征不明确，从轻工具开始。一旦出现 CAPTCHA、登录墙、内容需交互才可见、静态内容缺失等结构性信号，切换到浏览器自动化。

浏览器自动化前必须阅读：

```text
references/access/playwright-cli/guide.md
```

## 4. 人机协作边界

以下情况应请求用户介入：

- 需要登录、付费、订阅或授权，而当前没有可用会话。
- 多条路径都可行，但稳定性、速率限制、账号风险等取舍需要用户偏好。
- 已尝试命令库、轻工具和浏览器自动化，仍无法突破权限、反爬或结构变更。

以下情况应自行处理：

- 工具选择和工具切换。
- 网络波动、页面加载慢、单次临时失败。
- 能通过更换信息来源或执行路径绕过的问题。

请求用户介入时说明三点：

1. 当前已经完成什么。
2. 遇到什么障碍。
3. 建议用户选择什么具体方案。

## 5. 浏览器探索原则

使用浏览器自动化时，以完成当前信息获取为第一目标。

- 先快速判断目标内容是否在 DOM、页面状态、嵌入数据或接口响应中。
- 能用 `eval` 快速验证的选择器，不要先写复杂脚本。
- 需要交互时优先使用稳定选择器，不依赖临时 snapshot ref 作为长期依据。
- 不主动关闭或改动用户已有标签页；尽量在自己创建的标签页中操作并结束后关闭。
- 需要用户登录时，暂停自动化，让用户在浏览器中自行完成登录。

## 6. Capture Assessment

探索结束后，先交付用户请求的结果，再输出 Capture Assessment。

Capture Assessment 只判断是否存在后续 capture 候选，不创建 capture，不写 evidence，不生成命令包。

建议存在 capture 候选的条件：

- 本次实际跑通了稳定的信息获取路径。
- 该路径换参数即可复用。
- 能明确 domain/action。
- 能记录 URL/API/选择器/字段/输入输出等证据。
- 不与已有命令库能力明显重复。

不建议 capture 的常见原因：

- 本次只是一次性问答，没有稳定可参数化路径。
- 只使用搜索摘要，没有验证一手来源。
- 需要人工判断为主，自动化复用价值低。
- 目标路径依赖临时页面状态或高风险账号操作。
- 本次没有实际跑通，只是推测可能可行。

推荐输出格式见 `SKILL.md`。
