# Internationalization Documentation Maintenance

This project uses Chinese as the working language and English as the publish language. Agents must follow these conventions when modifying documentation.

## Documentation Layers

| Directory | Purpose | Audience |
|-----------|---------|----------|
| `src/explore/`, `src/compile/`, `src/access/` | AI reference docs | AI Agent |
| `docs/` | Human developer docs | Developers |
| `skills/websculpt/SKILL.md` | Skill entry | AI Agent |

## Naming Conventions

- Chinese source file: `foo.md`
- English translation: `foo.en.md`
- Both must be in the **same directory**

Example:

```
src/explore/
  strategy.md        # Chinese (the only manually editable source)
  strategy.en.md     # English (generated by Agent translation)
```

## Link Conventions (Critical)

Relative path links in `.en.md` files must be **identical** to those in `.md` files. The `.en.md` suffix must never appear.

- **Incorrect**: `[guide](../access/playwright-cli/guide.en.md)`
- **Correct**: `[guide](../access/playwright-cli/guide.md)`

Reason: when `scripts/build-skills.js` builds the English skill package, it renames all `.en.md` files to `.md`. If an English document contains `.en.md` links, they become dead links after the build.

## Synchronization Rules When Editing Documents

After modifying any Chinese `.md` file, you must complete the following in the **same session**:

1. **Incremental translation**: translate only changed paragraphs; do not re-translate the entire document.
2. **Link guarding**: ensure relative paths in `.en.md` are identical to those in `.md`.
3. **Ripple check**: if new references to external documents are added, confirm the referenced documents have corresponding `.en.md` files; if not, translate them as well.
4. **Docs synchronization**: if a file under `docs/` is modified, update the corresponding file under `docs/en/`.

## build-skills.js Dual-Package Behavior

`npm run build:skills` generates two skill packages:

- `skills/websculpt/`: Chinese version (copies src/ directly, preserving `.md`)
- `skills/websculpt-en/`: English version (copies src/, then overwrites `.md` with `.en.md` and deletes `.en.md`)

Agents do not need to manually intervene in the build process, but **must ensure links in `.en.md` do not carry the `.en` suffix**, otherwise the build output will contain dead links.

## Skill Install Language Selection

- `websculpt skill install`: installs the English version by default (from `skills/websculpt-en/`)
- `websculpt skill install --lang zh`: explicitly installs the Chinese version (from `skills/websculpt/`)

## Pre-Publish Checks

Run `npm run verify:i18n` to perform automated checks:

1. Every `.md` has a corresponding `.en.md`
2. No dead links with `.en.md` suffix in `.en.md` files
3. `docs/en/` and `docs/` files are one-to-one mapped
4. No residual `.en.md` files in the `skills/websculpt-en/` build output

**This check is integrated into `prepublishOnly`** and runs automatically before publishing; it blocks release if it fails.
