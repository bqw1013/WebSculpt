# 国际化文档维护

本项目采用中文作为工作语言，英文作为发布语言。Agent 在修改文档时必须遵循以下规范。

## 文档分层

| 目录 | 用途 | 读者 |
|------|------|------|
| `src/explore/`、`src/compile/`、`src/access/` | AI 参考文档 | AI Agent |
| `docs/` | 人类开发者文档 | 开发者 |
| `skills/websculpt/SKILL.md` | Skill 入口 | AI Agent |

## 命名规范

- 中文源文件：`foo.md`
- 英文翻译：`foo.en.md`
- 两者必须位于**同一目录**

示例：

```
src/explore/
  strategy.md        # 中文（唯一可手动编辑的源文件）
  strategy.en.md     # 英文（由 Agent 翻译生成）
```

## 链接规范（关键）

`.en.md` 文件中的相对路径链接**必须和 `.md` 文件完全一致**，不能出现 `.en.md` 后缀。

- **错误**：`[guide](../access/playwright-cli/guide.en.md)`
- **正确**：`[guide](../access/playwright-cli/guide.md)`

原因：`scripts/build-skills.js` 构建英文 skill 包时，会把所有 `.en.md` 重命名为 `.md`。如果英文文档里写了 `.en.md` 链接，构建后就会变成死链。

## 修改文档时的同步规则

当你修改任何中文 `.md` 文件后，**必须在同一次会话内**完成以下动作：

1. **增量翻译**：只翻译发生变更的段落，不要全文重翻。
2. **链接守护**：确保 `.en.md` 中的相对路径和 `.md` 完全一致。
3. **涟漪检查**：如果新增了对外部文档的引用，确认被引用的文档也存在对应的 `.en.md`；如果不存在，一并翻译。
4. **docs 同步**：如果修改了 `docs/` 下的文件，同步更新 `docs/en/` 下的对应文件。

## build-skills.js 双包行为

`npm run build:skills` 会生成两个 skill 包：

- `skills/websculpt/`：中文版（直接复制 src/，保留 `.md`）
- `skills/websculpt-en/`：英文版（复制 src/ 后，用 `.en.md` 覆盖同名 `.md`，并删除 `.en.md`）

Agent 不需要手动干预 build 过程，但**必须确保 `.en.md` 中的链接不带上 `.en` 后缀**，否则 build 产物会出现死链。

## skill install 语言选择

- `websculpt skill install`：默认安装英文版（从 `skills/websculpt-en/`）
- `websculpt skill install --lang zh`：显式安装中文版（从 `skills/websculpt/`）

## 发布前检查

运行 `npm run verify:i18n` 执行自动化检查：

1. 每个 `.md` 都有对应的 `.en.md`
2. `.en.md` 中没有 `.en.md` 后缀的死链
3. `docs/en/` 和 `docs/` 文件一一对应
4. `skills/websculpt-en/` 构建产物中没有残留的 `.en.md`

**该检查已集成到 `prepublishOnly`**，发布前会自动执行，不通过则阻止发布。
